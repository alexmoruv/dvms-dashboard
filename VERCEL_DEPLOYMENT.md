# 🚀 Развёртывание на Vercel (Безопасный вариант)

## ✅ Преимущества развёртывания на Vercel:

- ✅ **API ключи хранятся на сервере** — не видны в публичном коде
- ✅ **Автоматический деплой** при коммитах в GitHub
- ✅ **Бесплатно** для публичных проектов
- ✅ **Масштабируется** автоматически
- ✅ **HTTPS по умолчанию**
- ✅ **CDN для быстрой доставки**

---

## 🛠️ Этап 1: Получить API ключи

### 1. Gigachat API ключ (Сбер)
1. Перейдите на https://lk.sber.ru/
2. Компании → API → Gigachat
3. Скопируйте API ключ

### 2. Tavily API ключ (опционально)
1. Перейдите на https://app.tavily.com
2. Скопируйте API ключ

---

## 📦 Этап 2: Развёртывание на Vercel

### Шаг 1: Установить Vercel CLI
```bash
npm install -g vercel
```

### Шаг 2: Залогиниться в Vercel
```bash
vercel login
```

### Шаг 3: Развернуть проект
```bash
cd dvms-dashboard
vercel
```

Следуйте интерактивным подсказкам:
- **Link to existing project?** → No
- **Project name** → `dvms-dashboard` (или своё имя)
- **Which scope** → выберите личный аккаунт
- **Source code** → ./ (текущая папка)

### Шаг 4: Добавить переменные окружения

После успешного деплоя откроется ссылка на проект. Перейдите в **Settings**:

1. **Settings** → **Environment Variables**
2. Добавьте две переменные:

```
GIGACHAT_API_KEY = ваш_ключ_gigachat_здесь
TAVILY_API_KEY = ваш_ключ_tavily_здесь
```

3. Нажмите **Save**

### Шаг 5: Перепубликовать проект
```bash
vercel redeploy
```

Проект пересоберётся с новыми переменными окружения.

---

## 🌐 Этап 3: Подключить GitHub (Автоматический деплой)

### В Vercel Dashboard:

1. Нажмите **Settings** → **Git Integration**
2. Нажмите **Connect Git Repository**
3. Выберите GitHub
4. Авторизуйтесь и выберите `alexmoruv/dvms-dashboard`

**Результат:** Теперь при каждом пуше в `main` ветку, Vercel автоматически делает деплой.

---

## ✅ Проверить что всё работает

1. Откройте URL вашего проекта на Vercel (например, `https://dvms-dashboard.vercel.app`)
2. Перейдите на вкладку **Парсер**
3. Запустите парсинг данных

Если всё работает — мероприятия будут загружены! 🎉

---

## 🔐 Безопасность

### Что происходит:

1. **Публичный репо на GitHub** ← содержит только код, БЕЗ ключей
2. **Vercel хранит ключи** ← в защищённых переменных окружения
3. **Функции в `/api/` используют ключи** ← они видят переменные окружения только на сервере
4. **Клиент вызывает `/api/gigachat`** ← без отправки ключей

### Даже если кто-то скачает код:
- Ключи НЕ будут видны (они только на Vercel)
- Не сможет запустить без своих ключей

## 🆘 Решение проблем

### Ошибка: "GIGACHAT_API_KEY not configured"
✅ Проверьте что переменная добавлена в Vercel Settings → Environment Variables  
✅ После добавления нужна перепубликация (`vercel redeploy`)

### Ошибка 502 Bad Gateway
✅ Функция в `/api/` может быть медленной  
✅ Проверьте логи в Vercel Dashboard

### Приложение работает локально, но не на Vercel
✅ Может быть блокировка CORS  
✅ Проверьте что API функции возвращают правильные headers

## 💡 Локальное развитие

Пока работаете локально, вы всё ещё можете использовать `.env.local` с реальными ключами:

```bash
# .env.local (НЕ коммитить!)
VITE_GIGACHAT_API_KEY=sk-ваш_ключ
VITE_TAVILY_API_KEY=tvly-ваш_ключ

npm run dev
```

Приложение автоматически выберет между:
- **Локальными ключами** (если есть в .env для локального деплоя)
- **Vercel proxy** (если на продакшене или нет локальных ключей)
